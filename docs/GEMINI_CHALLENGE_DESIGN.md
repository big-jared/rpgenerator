# RPGenerator - Gemini Live Agent Challenge Design

## Vision

A living audiobook RPG. You speak, the world responds — with voice, art, and music.
No text boxes. No turn-based feel. Continuous, immersive, multimodal.

**New Adventure flow** (voice-driven):
- Gemini speaks: "What kind of world calls to you?"

- Gemini speaks: "Interesting... and who are you in this world?"
- Player describes their character naturally
- Gemini generates a character portrait as you speak

Maps to existing: `GameConfig`, `SystemType`, `CharacterCreationOptions`
New: Voice-driven instead of form-based. Gemini creates config from conversation.

- Music is underneath everything, shifting with emotional beats

**No visible text input.** Voice is primary. Optional keyboard for accessibility.

## Architecture

### Gemini Integration Layers

```
┌─────────────────────────────────────────────┐
│              Compose Multiplatform UI        │
│  (Scene art, subtitles, HUD, audio player)  │
└──────────────────┬──────────────────────────┘
                   │
┌──────────────────▼──────────────────────────┐
│           GeminiGameSession                  │
│  Manages Live API WebSocket connection       │
│  Routes audio in/out                         │
│  Handles tool calls from Gemini              │
└──────────────────┬──────────────────────────┘
                   │
        ┌──────────┼──────────┐
        ▼          ▼          ▼
   ┌─────────┐ ┌───────┐ ┌────────┐
   │ Live API│ │Gemini │ │Gemini  │
   │ (voice) │ │(image)│ │(music) │
   └────┬────┘ └───┬───┘ └───┬────┘
        │          │          │
        ▼          ▼          ▼
   Voice I/O   Scene art   Ambient
   streaming   generation  music gen
```

### How Voice + Tools Work Together

```
Player speaks: "I want to talk to the blacksmith"
        │
        ▼
Gemini Live API receives audio
        │
        ▼
Gemini calls tool: resolve_npc(name="blacksmith")
  → Returns: {npc: "Grimjaw", disposition: "friendly", context: "..."}
        │
        ▼
Gemini calls tool: get_game_state()
  → Returns: {location, quests, inventory, ...}
        │
        ▼  (NON_BLOCKING / WHEN_IDLE)
Gemini calls tool: generate_scene_image(prompt="player approaching forge")
  → Returns: image bytes → UI updates background
        │
        ▼
Gemini speaks AS the NPC: "Ah, another adventurer. Looking for steel?"
  (voice changes to match NPC character)
        │
        ▼
Player responds naturally, conversation continues...
```

### Tool Declarations for Gemini

```kotlin
// Core game tools exposed to Gemini Live API
val gameTools = listOf(
    // State queries
    Tool("get_player_stats") { game.getState().playerStats },
    Tool("get_inventory") { game.getState().inventory },
    Tool("get_active_quests") { game.getState().quests },
    Tool("get_npcs_here") { game.getState().npcsAtLocation },
    Tool("get_location") { game.getState().location },

    // Actions (modify game state)
    Tool("move_to_location", params: locationName) { orchestrator.movePlayer(it) },
    Tool("attack_target", params: targetName) { orchestrator.initiateCombat(it) },
    Tool("use_item", params: itemId) { orchestrator.useItem(it) },
    Tool("use_skill", params: skillId, target) { orchestrator.useSkill(it) },
    Tool("pick_up_item", params: itemName) { orchestrator.pickUpItem(it) },
    Tool("talk_to_npc", params: npcName) { orchestrator.startConversation(it) },

    // Quest management
    Tool("start_quest", params: questId) { orchestrator.startQuest(it) },
    Tool("update_quest_progress", params: questId, objectiveId, progress) { ... },
    Tool("complete_quest", params: questId) { orchestrator.completeQuest(it) },

    // World generation (Gemini decides when to create new content)
    Tool("generate_location", params: description) { orchestrator.generateLocation(it) },
    Tool("generate_npc", params: archetype, location) { orchestrator.generateNPC(it) },
    Tool("generate_quest", params: context) { orchestrator.generateQuest(it) },

    // Multimodal output triggers
    Tool("generate_scene_art", params: sceneDescription) { geminiImage.generate(it) },
    Tool("shift_music_mood", params: mood) { musicEngine.transition(it) },
)
```

### Key Difference from Current Architecture

**Current:** Multiple specialized agents (Narrator, GM, NPC, Quest) each with their own LLM session, orchestrated by code.

**New:** Single Gemini Live session IS the narrator/GM. It has tools to:
- Query game state (replaces internal state reads)
- Modify game state (replaces orchestrator routing)
- Generate content (locations, NPCs, quests via tools)
- Generate media (images, music via tools or native interleaved output)

The orchestrator logic moves FROM code routing TO Gemini's system prompt + tool selection.
Your game engine becomes the "backend" that Gemini calls into.

### System Prompt (Gemini Live Session)

```
You are the narrator and game master of a LitRPG adventure.
You speak to the player in an immersive, atmospheric voice.

WORLD: {worldSeed.description}
NARRATOR STYLE: {worldSeed.narratorPrompt}
PLAYER: {characterSheet summary}
LOCATION: {currentLocation}
NPCS HERE: {npcList}
ACTIVE QUESTS: {questSummary}

RULES:
- Narrate continuously — describe scenes, reactions, atmosphere
- When the player speaks, respond naturally and take action via tools
- Use generate_scene_art when the visual scene changes significantly
- Use shift_music_mood when emotional tone shifts
- Voice NPCs with distinct personalities when player talks to them
- Call game state tools to stay grounded in actual game data
- Never hallucinate stats — always query get_player_stats
- Combat: use attack_target tool, then narrate the result dramatically
- Keep the player engaged — suggest possibilities without railroading
```

## New GameEvent Types

```kotlin
sealed class GameEvent {
    // Existing
    data class NarratorText(val text: String) : GameEvent()
    data class NPCDialogue(...) : GameEvent()
    data class SystemNotification(...) : GameEvent()
    data class CombatLog(...) : GameEvent()
    data class StatChange(...) : GameEvent()
    data class ItemGained(...) : GameEvent()
    data class QuestUpdate(...) : GameEvent()

    // New multimodal events
    data class SceneImage(val imageBytes: ByteArray, val description: String) : GameEvent()
    data class NarratorAudio(val audioChunk: ByteArray) : GameEvent()
    data class MusicChange(val mood: String, val audioBytes: ByteArray?) : GameEvent()
    data class NPCPortrait(val npcId: String, val imageBytes: ByteArray) : GameEvent()
    data class AmbientSound(val soundType: String, val audioBytes: ByteArray) : GameEvent()
}
```

## What Makes This Win

1. **Not a chatbot** — it's a game. Stats, quests, combat, progression.
2. **Voice-native** — built for Live API, not text with TTS bolted on.
3. **Interleaved output done right** — scene art evolves mid-narration, music shifts with mood.
4. **Real game engine** — Gemini calls into actual game logic via tools, not just freeform storytelling.
5. **Full Kotlin stack** — Compose Multiplatform + google-genai Java SDK. Google's own ecosystem.
